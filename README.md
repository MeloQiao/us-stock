## 策略详解

所有策略的输出统一为：`+1`（买入）、`-1`（卖出/平仓）、`0`（观望）。

### 趋势类策略

趋势策略的核心假设：**价格会持续沿当前方向运动**。适合单边市场，在震荡市会反复被打止损。

---

#### 1. 黄金交叉 Golden Cross

**原理**

用两条不同周期的移动平均线（MA）来判断趋势方向。短期 MA 上穿长期 MA 为金叉（买入信号），下穿为死叉（卖出信号）。

```
短期MA（20日）> 长期MA（100日） → BUY +1
短期MA（20日）< 长期MA（100日） → SELL -1
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `fast` | 20 | 短期移动平均线周期（天） |
| `slow` | 100 | 长期移动平均线周期（天） |

**参数选择逻辑**

经典黄金交叉用 50/200，但 15 年回测显示 **20/100 在美股优于 50/200**，原因是美股主要趋势周期在 1-4 个月（20-100交易日），50/200 太滞后，经常信号出来时趋势已经走了一大半。

**权重：2.0**（最高）——历史信号质量最稳定，误判少，是系统的核心指标。

---

#### 2. 超级趋势 Supertrend

**原理**

基于 ATR（Average True Range，平均真实波幅）动态计算上下轨，价格突破上轨为下降趋势，突破下轨为上升趋势。相比固定 MA，Supertrend 能根据波动率自动调整灵敏度——高波动时轨道更宽，低波动时更窄。

```
上轨 = (High + Low) / 2 + multiplier × ATR
下轨 = (High + Low) / 2 - multiplier × ATR

价格 > 上轨 → 上升趋势 → BUY +1
价格 < 下轨 → 下降趋势 → SELL -1
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `atr_period` | 10 | 计算 ATR 的周期（天）。越小越灵敏，越大越稳定 |
| `multiplier` | 2.5 | 轨道宽度倍数。越小止损越紧（容易被振出），越大止损越宽（回撤更大） |

**参数选择逻辑**

`multiplier=2.5` 是回测最优解（原始值 3.0 偏宽）。2.5 在趋势反转时更快出场，控制最大回撤效果明显。

**权重：1.0**

---

#### 3. 唐奇安通道 Donchian Channel

**原理**

20日最高价构成上轨，10日最低价构成下轨。价格突破20日高点为做多信号（突破型策略，源自"海龟交易法则"）。

```
价格 > 最近20日最高价 → BUY +1  （上轨突破，新高动能）
价格 < 最近10日最低价 → SELL -1 （下轨跌破，趋势反转）
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `entry_period` | 20 | 买入突破的回望窗口（天）。突破20日新高才买 |
| `exit_period` | 10 | 卖出的回望窗口（天）。跌破10日新低才卖 |

**参数选择逻辑**

入场宽（20天）出场快（10天）：让利润奔跑，但一旦回撤超过 10 天的低点就离场。回测结果显示这组参数 15 年内无需调整，本次优化也确认维持不变。

**权重：1.0**

---

#### 4. EMA + ADX 趋势过滤

**原理**

两个指标组合使用：
- **EMA（指数移动平均线）**：比普通 MA 对近期价格更敏感
- **ADX（Average Directional Index，平均趋向指标）**：衡量趋势强度，不判断方向。ADX > 20 表示趋势强，ADX < 20 表示震荡市

```
短期EMA > 长期EMA  且  ADX > 20  → BUY +1  （趋势向上且趋势强）
短期EMA < 长期EMA               → SELL -1
ADX < 20 时不给出买入信号（震荡市不进场）
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `ema_fast` | 8 | 短期 EMA 周期。越小越灵敏 |
| `ema_slow` | 21 | 长期 EMA 周期。经典"月线" |
| `adx_period` | 14 | ADX 计算周期（标准值） |
| `adx_threshold` | 20 | ADX 门槛。低于此值不给买入信号（认为是震荡市） |

**参数选择逻辑**

从 12/26 改为 8/21，门槛从 25 降到 20。更快的 EMA 在趋势初期更早入场；更低的 ADX 门槛意味着即使趋势刚刚形成也参与。

**权重：0.5**（最低）——信号频率过高，误判相对多，历史表现是9个策略中最弱的，因此权重减半。

### 动量类策略

动量策略的核心假设：**涨的还会继续涨，跌的还会继续跌**（短期价格惯性）。

---

#### 5. MACD 金叉死叉

**原理**

MACD 是最经典的动量指标，由三部分组成：
- **MACD线** = 12日EMA - 26日EMA（快慢均线之差）
- **信号线** = MACD线的9日EMA（MACD的移动平均）
- **柱状图** = MACD线 - 信号线

```
MACD线 上穿 信号线 → BUY +1   （动量转正）
MACD线 下穿 信号线 → SELL -1  （动量转负）
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `fast` | 12 | 快线 EMA 周期（标准值） |
| `slow` | 26 | 慢线 EMA 周期（标准值） |
| `signal_period` | 9 | 信号线平滑周期（标准值） |

**参数选择逻辑**

12/26/9 是全球最广泛使用的参数，15年回测结果确认这就是最优解，无需调整。

**权重：1.0**

---

#### 6. ROC 变动率动量

**原理**

ROC（Rate of Change）直接衡量价格在过去 N 天的涨幅。用两个周期对比：短期动量强于长期动量才进场。

```
短期ROC（20日涨幅）> 0  且  长期ROC（60日涨幅）> 0  → BUY +1
短期ROC < 0  或  长期ROC < 0                         → SELL -1
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `period_short` | 20 | 短期变动率周期（约1个月）|
| `period_long` | 60 | 长期变动率周期（约3个月）|

**参数选择逻辑**

短期确认近期动量，长期确认中期趋势，两者同向才入场，避免在趋势末尾追高。15年回测无需调整。

**权重：1.0**

### 均值回归类策略

均值回归策略的核心假设：**价格偏离均值太远终会回归**。适合震荡市，与趋势策略天然对冲，提升组合的全天候适应性。

> ⚠️ 这两个策略**不用于杠杆ETF**（TQQQ/SOXL等），因为杠杆产品有每日价值损耗，不适合均值回归逻辑。

---

#### 7. RSI 超买超卖

**原理**

RSI（Relative Strength Index，相对强弱指标）衡量最近一段时间内上涨天数与下跌天数的比例，值域 0-100。

```
RSI < 25  → 超卖，价格被过度打压  → BUY +1
RSI > 75  → 超买，价格被过度推高  → SELL -1
25 ≤ RSI ≤ 75                   → 持有不动
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `period` | 14 | 计算周期（标准值，约2-3周） |
| `oversold` | 25 | 超卖阈值。低于此值才是真正超卖（原30太宽松） |
| `overbought` | 75 | 超买阈值。高于此值才是真正超买（原70太宽松） |

**参数选择逻辑**

从 30/70 改为 25/75，要求更极端的超卖/超买才触发信号。减少了假信号（轻微回调不触发），牺牲了信号频率换取信号质量。

**权重：1.0**

---

#### 8. 布林带挤压 Bollinger Squeeze

**原理**

布林带由三条线组成：中轨（20日MA）+ 上轨（+1.5σ）+ 下轨（-1.5σ）。

本策略专注"带宽挤压"（Squeeze）现象：
- 当布林带变窄（低波动整理期），往往预示着即将爆发大行情
- 挤压结束后价格向上突破中轨 → 做多

```
带宽 < squeeze_threshold（0.10）       → 处于挤压状态
价格 > 中轨  且  刚结束挤压              → BUY +1
价格 < 中轨  且  刚结束挤压              → SELL -1
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `period` | 20 | 布林带计算周期（标准值） |
| `std_dev` | 1.5 | 标准差倍数。1.5 比标准的 2.0 更紧，挤压触发更频繁 |
| `squeeze_threshold` | 0.10 | 带宽/中轨的比值低于此值才认为是挤压状态 |

**参数选择逻辑**

从 2.0 改为 1.5，带宽更窄，squeeze 更容易被识别，信号频率提升。回测显示 1.5 在中等波动的成长股上效果最好。

**权重：1.0**

#### 9. VIX 恐慌指数择时

**原理**

VIX（CBOE 波动率指数）衡量市场对未来30天的隐含波动率预期，俗称"恐慌指数"。

```
VIX > 30  → 市场极度恐慌，通常是底部区域  → BUY +1
VIX < 15  → 市场过于乐观，可能见顶        → SELL -1
15 ≤ VIX ≤ 30                            → 中性，不给信号
```

**对反向ETF（SQQQ/SOXS）逻辑翻转：**
```
VIX > 30  → 市场恐慌，做空ETF此时已涨很多  → SELL -1（止盈）
VIX < 15  → 市场乐观，适合买入做空ETF     → BUY +1
```

**当前参数**
| 参数 | 值 | 含义 |
|------|----|------|
| `fear_threshold` | 30 | VIX 超过此值视为恐慌 |
| `greed_threshold` | 15 | VIX 低于此值视为贪婪 |

**权重：0.3**（第二低）——VIX 策略在低波动率长牛市环境中长期没有触发机会，历史贡献较弱，因此权重大幅降低。但保留它是为了在市场崩溃时提供额外的买入信号（2020年3月这类情况）。

### 综合评分系统

**工作原理**

9个策略的信号（+1/0/-1）乘以各自权重后求和，得到综合评分：

```
综合评分 = Σ (策略信号 × 策略权重)

最高分 = 2.0+1.0+1.0+0.5+1.0+1.0+1.0+1.0+0.3 = 8.8（全部看多）
最低分 = -8.8（全部看空，杠杆ETF不含均值回归两项则 ±6.8）

BUY  信号：综合评分 ≥ 6.0（美股/A股）或 ≥ 7.0（港股）
SELL 信号：综合评分 ≤ -3.0
HOLD：其余区间
```

**为什么港股阈值更高（7.0 vs 6.0）？**

港股受内地资金流向、南下通道、政策风险影响，信号噪音比美股更大。提高阈值要求更强的多方共识才入场，减少假突破。

**策略权重说明**

| 策略 | 权重 | 原因 |
|------|------|------|
| golden_cross | 2.0 | 15年回测信号质量最高，误判少 |
| supertrend | 1.0 | 稳定，不过度交易 |
| donchian_channel | 1.0 | 突破型，与趋势跟踪互补 |
| ema_adx | 0.5 | 信号过于频繁，误判相对多 |
| macd_crossover | 1.0 | 经典动量指标，可靠 |
| roc_momentum | 1.0 | 双周期确认，质量稳定 |
| rsi_strategy | 1.0 | 均值回归代表，补充趋势短板 |
| bollinger_squeeze | 1.0 | 捕捉爆发行情，与趋势策略互补 |
| vix_timing | 0.3 | 长牛市贡献弱，保留危机识别功能 |

## 市场机制识别（Regime）

系统每天根据基准指数（美股=SPY，港股=盈富基金，A股=沪深300）判断当前市场机制：

```
SPY 价格 > MA200  且  SPY 价格 > MA50  →  bull_strong（牛市强势）
SPY 价格 > MA200  且  SPY 价格 < MA50  →  bull_caution（牛市承压）
SPY 价格 < MA200                       →  bear（熊市）
```

**Regime 对交易的影响：**

| 机制 | 新建多单 | 反向ETF买入 | 单仓上限 |
|------|---------|------------|---------|
| bull_strong | ✅ 正常 | 🚫 不触发 | 30% |
| bull_caution | ✅ 正常 | 🚫 不触发 | 20% |
| bear | 🚫 屏蔽 | ✅ 允许 | 15% |

熊市下所有普通多单买入信号被屏蔽，但 SQQQ/SOXS/07552 这类反向ETF的买入信号允许通过。

## 仓位管理

**动态仓位公式：**

```
单标的最大仓位 = 总资金 × 基础比例 × 评分系数

基础比例：bull_strong=30%  bull_caution=20%  bear=15%
评分系数：clamp(综合评分 / 最大可能分, 0.5, 1.0)
```

**例：** 牛市承压，NVDA 综合评分 7/8.8
```
单仓上限 = 总资金 × 20% × (7/8.8) = 总资金 × 15.9%
```
评分越高，允许的仓位越大；评分只达到及格线，仓位打折。

**组合层面还有两道过滤：**
- 相关性去重：同批买入信号中两标的相关系数 > 0.85，保留评分更高的那个
- 行业集中度：单一行业不超过总仓位 65%

## 三层进阶防护系统

在 9策略综合信号之上，叠加三层保护逻辑（仅美股）。三层各自输出仓位系数，最终**相乘**作为对信号的综合缩放：

```
最终仓位规模 = 综合信号仓位 × CrashShield系数 × ML系数 × DualMomentum系数
```

---

### Layer 1：崩盘护盾 Crash Shield

**文件：** `strategies/crash_shield.py`

**逻辑：** 4个宏观子信号同时打分，累计分数决定防御级别。

| 子信号 | 触发条件 | 监测什么 |
|--------|---------|---------|
| VIX 恐慌加速 | VIX > 25 且 5日涨幅 > 20% | 市场恐慌骤升 |
| 趋势破坏 | 价格 < MA50 且 MA50 本身下行 | 中期趋势已坏 |
| 短期急跌 | SPY 20日跌幅 < -8% | 快速崩溃式下跌 |
| 信用利差恶化 | HYG 20日跌幅 < -3% | 信用市场为衰退定价 |

**防御级别：**

| 分数 | 级别 | 新买入 | 说明 |
|------|------|--------|------|
| 0–1 | NONE 🟢 | ×1.0 | 正常操作 |
| 2 | CAUTION 🟡 | ×0.5 | 新仓减半 |
| 3–4 | SHIELD 🔴 | ×0.0 | 屏蔽全部新多单 |

> HYG（iShares 高收益债 ETF）是信用市场的晴雨表。2008年 HYG 跌幅远早于股市，是绝佳的领先指标。

---

### Layer 2：ML 制度分类器

**文件：** `strategies/ml_regime.py`，训练脚本：`scripts/train_ml_regime.py`

**逻辑：** 用 XGBoost 模型预测「SPY 未来 20 天涨跌概率」，输出 P(上涨) ∈ [0, 1]，转换为仓位系数。

**特征（19个）：**
```
SPY  → 价格/MA20/MA50/MA200 比率、1M/3M/6M/12M 动量、
        从高点回撤幅度、21日/63日实现波动率及其比值、RSI
VIX  → 当前水平/相对MA20偏差/过去1年百分位/5日变化率
HYG  → 20日收益率、相对MA50偏差
```

**仓位系数映射：**
```
P(上涨) ≥ 0.45  →  系数 = 1.0（中性/看多，不干预）
P(上涨) = 0.30  →  系数 ≈ 0.75（偏空，轻减仓）
P(上涨) = 0.00  →  系数 = 0.25（极度看空，大幅减仓）
```

设计原则：**不确定时不干预**。只有当模型明确看空（概率 < 0.45）时才减仓，避免中性信号误伤牛市仓位。

**训练方式：Walk-Forward（无前瞻偏差）**
```
Window 1: 训练 2005–2014 / 验证 2015–2017
Window 2: 训练 2005–2017 / 验证 2018–2020
Window 3: 训练 2005–2020 / 验证 2021–2022
Window 4: 训练 2005–2022 / 验证 2023–今
```
每个窗口独立训练，只预测它从未见过的数据，OOS 结果真实可信。

**自动化：** 每月2日 06:00 UTC 自动重训练 + 上传 HF Dataset，当日 pipeline 使用新模型。

### Layer 3：双动量过滤

**文件：** `strategies/dual_momentum.py`

两个独立的动量检查：

**A. 绝对动量（市场择时）**
```
SPY 12个月收益率 > 0  →  允许做多（×1.0）
SPY 12个月收益率 ≤ 0  →  屏蔽所有新多单（×0.0）
```
当整体市场处于长期下行趋势时，不管个股信号多强，都不开新多头仓位。

**B. 动量崩溃保护（Momentum Crash Protection）**
```
21日实现波动率 ÷ 126日实现波动率 ≥ 2.0  →  减半仓位（×0.5）
```
短期波动率突然飙升到中期水平的 2 倍以上，是动量崩溃的典型特征（如 2008年11月、2020年3月）。

**C. 截面动量（选股排名）**
在有买入信号的标的中，按 6 个月收益率排名：
- 前 1/3（高动量）→ 权重 ×1.15
- 后 1/3（低动量）→ 权重 ×0.60
- 中间 → 权重 ×1.0

### v2 信号优化（Direction 1 + Direction 2）

> 基于回测发现"二元开/关"逻辑牺牲了大量"中等评分"时段的收益，因此引入以下两项改进：

#### Direction 1：分级仓位规模

|---------|---------|------|

| 2.5 – 4.5 | **25%** | 弱信号，轻仓试探 |
| -5.0 – 2.5 | **持有** | 中性区，保持当前仓位不动 |

| 方向 | 旧阈值 | 新阈值 | 效果 |

### Direction 3：修正感知 ML 标签（实验）

> **结论先行：不推荐部署。** Direction 1+2 已足够，Direction 3 在 15yr SPY 数据集上净效果为负，维持 v2 配置即可。

**设计动机**

v1 ML 训练标签仅以「未来 20 天收益 ≥ 0」作为正例，存在一个盲点：中期修正中的短暂反弹——例如 2022 年熊市期间，每次小反弹日（20d 后仍正收益）都被标为 y=1（看多），导致模型在整个 2021-2022 区间输出偏高的看多概率，ML 层对大级别回调的识别能力偏弱。

**v3 修正感知标签**

在 y=0（看空）的判断条件中额外增加"当前处于修正期"维度：

```python
rolling_peak = spy.rolling(252, min_periods=50).max()
dd_from_peak  = spy / rolling_peak - 1
in_correction = dd_from_peak < -0.08      # 距 1 年高点已跌超 8%
bearish       = (fwd < 0) | in_correction  # 短期跌 OR 处于中期回调
y = 1 if not bearish else 0
```

2015-16 调整、2018 Q4、2020 COVID、2022 熊市期间，即使某日 20d 后反弹，只要价格仍在年高点 -8% 以下，均标记为负例。

**训练标签对比（2011–2026，SPY）：**

| 版本 | 看空样本比例 | 变化 |
|------|------------|------|
| v1（原始标签） | 32% | 基准 |
| v3（修正感知） | **42%** | **+10pp，更多负例** |

**OOS 预测概率（Walk-Forward 窗口 3 对比最关键）：**

| 验证窗口 | v1 mean_prob | v3 mean_prob | 说明 |
|---------|-------------|-------------|------|
| 2015–2017 | 约 0.65 | 约 0.64 | 无显著差异 |
| 2018–2020 | 约 0.55 | 约 0.52 | 轻微偏空 |
| **2021–2022** | **0.530** | **0.236** | v3 大幅偏空（-29pp） |
| 2023–今 | 约 0.58 | **0.580** | 回归中性看多 |

v3 在 2021-2022 验证窗口把看多概率从 53% 压至 24%。但 2021 年是 SPY 强力反弹年（+28%），这导致 ML 乘数大幅压低了 2021 年的仓位，错过了实际涨幅。

## 自动优化机制

系统有两层独立的自动优化，每层优化不同的参数：

### 每周日：策略权重优化

`optimize_weights.yml` 每周日 02:00 UTC 自动运行。

基于过去 5 年历史数据，计算每个策略在不同 Regime 下的信号质量：
```
质量分 = 命中率 × (1 + 平均收益率 × 10)
```
质量高的策略权重提升，质量差的权重降低，限制在 [0.3, 3.0] 范围内。

结果保存到 HF Dataset，下次 daily pipeline 直接使用新权重。飞书会通知哪些策略权重有显著变化（Δ ≥ 0.15）。

### 每月1号：策略参数优化

`optimize_params.yml` 每月1日 05:00 UTC 自动运行。

对每个策略的内部参数做网格搜索（MA周期/ATR倍数/RSI阈值等），使用 **3段 Walk-Forward 验证**（训练8年→测试3年，滚动前进）评估样本外 Sharpe。

找到最优参数后：
1. 保存到 HF Dataset（带月份时间戳存档）
2. 自动修改 `config.py`
3. 自动 commit + push 到 GitHub
4. Streamlit Cloud 自动重新部署

飞书通知改了哪些参数以及样本外 Sharpe。

**注意：** 两层优化相互独立，参数优化和权重优化操作不同的配置维度，不会冲突。

## 部署与运维

| 组件 | 平台 | 触发时间 | 说明 |
|------|------|---------|------|
| 前端 UI | Streamlit Community Cloud | push 触发 | 自动检测 main 分支变更重新部署 |
| 日线 pipeline | GitHub Actions | 美股21:30 / 港股08:30 / A股07:30 UTC | 拉数据→信号→纸交易→飞书 |
| 权重优化 | GitHub Actions | 每周日 02:00 UTC | 基于5年数据重新计算各策略权重 |
| 参数优化 | GitHub Actions | 每月1日 05:00 UTC | Walk-Forward网格搜索，自动push config.py |
| **ML模型重训练** | **GitHub Actions** | **每月2日 06:00 UTC** | **XGBoost重训练，上传 HF Dataset** |
| 数据存储 | Hugging Face Dataset | 每日写入 | OHLCV / 信号 / 持仓 / 权重 / 参数 / ML模型 |
| 纸交易（美股）| Alpaca Paper Trading | 每日自动 | 系统自动下单，可在 Robinhood 手动跟单 |
| 纸交易（港股/A股）| 虚拟持仓 | 每日自动 | 存储于 HF Dataset，无真实下单 |
| 告警推送 | 飞书自定义机器人 | 每日收盘后 | 含三层防护状态，关键词过滤：含"Stock" |

**完全自动化，零人工干预。** 所有优化任务均由 GitHub Actions 在服务器执行，结果自动保存并应用。
